Detailed Description

These functions are used to initialize the core and to update it.

  • lui_init() is only called once at the very beginning.
  • lui_update() is called periodically in an infinite loop.

◆ lui_init()

int8_t lui_init ( uint8_t  mem_block[],
uint16_t  size 
)

Initialize the LameUI core. Here user provides LameUI some RAM to create widgets. If and when RAM is not sufficient, widget creation will fail.

If init fails due to memory allocation problem, returns -1, else returns 0.

Parameters
mem_blockan array of uint8_t. this will be used as work area
sizesize of the alloted memory
Returns
0: Success, -1: Fail

◆ lui_update()

void lui_update ( )

This function updates the UI by reading input and rendering widgets that need rendering.

User must call this function periodically (e.g. every 20ms) in a loop so no input event is missed. Also, LameUI renders only when widgets need rendering. So, draw calls are not wasted.
